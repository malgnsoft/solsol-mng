## D. ⚠️ 정본 상충 — 에스컬레이션 (문서 무수정, SoT 소유자 조정 요망)
- **쟁점**: 강사/서브강사 `operation`(운영) 메뉴 권한이 **05 M-2 표 ↔ 06 §2.8**에서 불일치.
  - 05 M-2 표: 강사 운영 △(허용, own) / 결정 본문(§M-2 규칙 #3): "서브강사 = 강사 − 운영·정산·사용자관리".
  - 06 §2.8: 강사 허용목록에 operation **없음** / 서브강사 operation **차단**.
- **총괄 판정**: 권위 우선순위(**05 정책설계서 확정 > 06 API계약**)에 따라 **코드는 05를 정본으로 따름** — 강사 operation 허용(own), 서브강사 operation 차단(=강사−운영). 현 `rbacPreset.ts`가 이에 부합.
- **요청**: 06 §2.8 강사 허용목록에 `operation(own)` 반영 여부를 **강테크·임기획**이 확정(06을 05에 정렬). → `docs/validation/`은 읽기 전용이라 본 앱에서 수정하지 않음.

## E. Phase 1 서버 이관 필수 (배포 게이트 — 미해결 시 '상' 승격)
> 이 슬라이스는 UI 프리셋·모달일 뿐 **실보안 0** — 아래 서버 구현 없이는 권한경계 부재.
1. PUT permissions 서버검증(요청자 owner 인가·role별 화이트리스트/스코프 상한 강제·owner 조합 거부).
2. roleGuard 미들웨어(menu_key 인가·직접URL 403 EXC-06).
3. 데이터스코프 own 서버 강제(owner_admin_id=본인 필터).
4. 세션 서버화(HttpOnly 쿠키)·클라 자격증명 비교/localStorage owner 복원 제거.
5. 초대 토큰 서버발급(암호난수·48h·1회성·레이트리밋).
6. 정산/지급 계좌 **응답 단계 마스킹**(원본 평문 미하달).
7. (선택) C-16 트리 "알림보기"(P-AD-11 알림센터 접근) 노드 편입 검토.
