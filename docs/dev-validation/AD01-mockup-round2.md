## B. 통합 결함표 — blocker 테마 (중복 제거·근본원인 기준)

10개 게이트의 '상' 결함을 **8개 근본원인 계열**로 묶음. 각 계열은 여러 화면에 걸침.

### ■ B-1. PII 마스킹 유틸 전면 부재 〔privacy 핵심 · 상〕
공통 마스킹 컴포저블(이메일/휴대폰/계좌/사업자번호)이 **존재하지 않아** 관리자단 목록·상세가 전부 원문 렌더. 관리자 화면은 "본인 조회"가 아니므로 C-6 비마스킹 예외 대상 아님(P-AD-17·CMP-01·표준 §7).
- 학습자 목록/상세 이메일·휴대폰 평문 · 강사 상세 이메일·휴대폰 평문 · 관리자 목록/상세(중) · **마케팅 발송내역 상세 수신자 휴대폰 대량 평문** · 환불 상세 이름+이메일 평문 · 정산 사업자번호/계좌 실패턴 미적용 · 문의 작성자 실명.
- **제안**: `useMask()`(email `mi****@`, phone `010-****-5678`, account `000-***-000000`) 유틸 1개 도입 → 전 목록·상세 일괄 적용. 목업 단계에 유틸을 넣어 구현 이관 시 누락 방지.

### ■ B-2. RBAC(05 확정 #3 = M-2) 골격 전면 미반영 〔security 핵심 · 상〕
- 사이드바(`AdminSidebar.vue`) **역할 무관 고정 배열**(11메뉴 전량 노출) — hide/menuKey/role 분기 없음.
- 라우트 가드 미들웨어 **전무** → 모든 `/admin/**` 직접 URL 무권한 접근(EXC-06 위반).
- 로그인 role이 **클라이언트 라디오 자칭**(`ref('admin')`), 서버 강제 없음. 서브강사 개념 부재.
- C-16 권한 모달이 **평면 체크박스**(강사6/관리자10 임의) — SoT의 2-step 체크트리(상위→하위 연동)+강사추가 서브단계+역할 default 프리셋과 불일치.
- **제안**: role별 화이트리스트 상수화 → LNB hide, C-16을 공유 컴포넌트로 재구성(M-2 매트릭스·서브강사), 실연동 시 `auth`/`rbac` 미들웨어. **role은 서버 응답에서만 확정**.

### ■ B-3. 비가역 행위 게이트 부재 〔security·privacy·sales 공통 · 상〕
환불 승인/거부, 학습자 탈퇴, 일괄 삭제가 **컨펌·경고 없이 원클릭**(CMP-06 위반).
- 환불 상세 승인/거부 무컨펌 · 학습자 일괄 탈퇴(1명씩·강좌보유 차단·최종컨펌 없음) · 마케팅/팝업/환불/페이지 "일괄 삭제" 무컨펌 · 크레딧 취소(미사용&7일 조건·컨펌 없음).
- **제안**: 공통 컨펌 모달 패턴(금액/영향 재확인+"되돌릴 수 없습니다") 1개로 통일 적용.

### ■ B-4. 인증 화면 정책 위반 〔intro · 상〕
- 회원가입 **닉네임 필드 자체 부재**(05 C-1 2~15자 위반) + **역할 라디오(강사/관리자 default 강사) 부재**.
- 로그인 **default 역할이 관리자**(SoT=강사).
- 가입 완료 버튼 카피 "로그인 하기"(오기) · restricted 본문이 "쏠쏠 문의/사이트관리 바로가기" 대신 **크레딧 충전 서사로 오구현**.
- 비밀번호 문구 05 C-3("영문·숫자·특수문자 3종 8~16자") 미준수(가입·재설정), 가입코드 타이머 정적 `04:59`(≠10분 C-2).
- 내 정보 닉네임 필드 부재(C-1).

### ■ B-5. 정책 정합 광범위 불일치 (상태머신·라벨·계산식·경계값) 〔전 클러스터 · 상/중〕
목업이 SoT 개정 정책ID를 반영하지 못한 라벨·값 오정합 다발.
- **정산 계산식 P-AD-110 구조 불일치**(PG수수료 컬럼 부재·원천세 임의추가·부가세 산식 오류 → **금전 표기 오류**).
- 캠페인 발송 **상태머신 라벨 SoT 5종 불일치**(진행/대기/종료/임시/중지) · 환불 상태 3종↔4종 · 주문 상태머신(유예/미결제 누락) · 커뮤니티 판매상태 "판매종료"→"판매마감" · 커뮤니티 구독료 상한 9,990,000→**9,999,999**.
- KPI 명칭 다수 불일치(학습자·캠페인·강의 목록), 옵션/채널/노출대상 종수 불일치(플레이어 4종·알림 2채널·공지 4대상·팝업 노출규칙 등).

### ■ B-6. 상품/콘텐츠/운영 **핵심 요소 미구현** 〔products·contents·ops · 상〕
- **라이브/화상 상세 탭**(자료실·수료·학습자·결제·후기) 전부 "API 연동 필요" placeholder · 입장버튼 활성화시간 셀렉트 누락.

## E. 수정 우선순위 (목업 수정 착수용)

목업 수정 시 **공통 인프라(P0) → 정책 정합(P1) → 디테일(P2)** 순 권장. P0는 여러 화면에 파급되는 근본원인이라 먼저 처리 시 결함 다수 동시 해소.

- **P0 (공통 인프라 — blocker 대량 동시 해소)**
  1. **마스킹 유틸**(B-1) 1개 → 전 목록·상세 적용.
  2. **비가역 컨펌 모달**(B-3) 공통 패턴 → 환불·탈퇴·삭제·크레딧 취소.
  3. **RBAC 골격**(B-2) — 사이드바 role 화이트리스트 hide + C-16 공유 컴포넌트(트리·프리셋·서브강사) + (실연동)미들웨어 스텁.
  4. **공통 상태 컴포넌트**(B-8) — Empty/Skeleton/Error + 페이지네이션.
- **P1 (정책 정합 — 화면별)**
