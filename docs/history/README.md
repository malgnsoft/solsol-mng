# 작업 이력 (`doc/history/`)

날짜별 작업 내역을 누적해 두는 폴더입니다. **하루에 한 파일** 원칙.

## 파일명 규칙

```
history.yyyyMMdd.md
```

예: `history.20260514.md`. 작업이 있는 날만 생성합니다.

## 파일 구조 (공통)

각 파일은 다음 순서를 따릅니다.

1. **한 줄 요약** — 그 날의 가장 큰 변화
2. **번호별 작업 섹션** — 결정 사항 / 코드 변경 / 배포 / 미세 조정
3. **산출물** — 추가·수정된 파일, 배포 버전, 외부 URL
4. **다음 단계 / 알려진 한계**
