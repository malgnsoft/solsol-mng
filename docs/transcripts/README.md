# transcripts — 세션 대화·에이전트 소통 기록 (일단위)

각 개발 세션([`DEV_SESSION_PROTOCOL.md`](../DEV_SESSION_PROTOCOL.md))의 **사용자 프롬프트**와 **에이전트 간 소통(총괄↔팀장↔팀원)** 을 **하루 단위로** 기록한다.

## 구조 (클로버 방지 — 세션별 파일 분리)

```
docs/transcripts/
  README.md
  <YYYYMMDD>/
    INDEX.md            ← 그날 세션 목록·요약 (허브 관리)
    hub.md              ← 허브 (프로젝트 관리, solsol-mng)
    solsol.md           ← 쏠쏠 (풀스택 + solsol_lms 테넌트 스키마)
    solsol-brand.md     ← 쏠쏠 브랜드 (풀스택 + solsol master 스키마)
```

> 세션은 **3개(Phase A)**: `hub`·`solsol`·`solsol-brand`. (2026-07-02엔 한시적으로 `backend-db` 세션이 데이터 정본 단일화를 수행 → 이후 제품/허브로 흡수. 그날 `backend-db.md`는 역사 기록으로 보존.)

- 세션들은 같은 파일시스템에서 동시에 돈다 → **한 파일을 여러 세션이 편집하면 덮어쓴다**. 그래서 **세션마다 자기 파일**에만 append 한다.
- **각 세션이 자기 파일을 소유·기록**(그 세션이 자기 프롬프트·소통을 가장 정확히·안전하게 요약할 수 있다). 허브는 `INDEX.md`와 `hub.md`를 관리한다.

## 파일 형식 (세션별)

```
# <세션명> 트랜스크립트 — YYYY-MM-DD

## 1. 사용자 프롬프트 (사용자 → 세션) — **원문 그대로**
> 사용자가 입력한 텍스트를 **한 글자도 바꾸지 말고** 인용 블록으로 붙인다(요약 금지).
```
① [원문 붙여넣기]
```

## 2. 에이전트 소통 (총괄 ↔ 팀장 ↔ 팀원) — **원문 그대로**
> 총괄이 팀원에게 보낸 **위임 브리프(프롬프트) 전문** + 팀원이 반환한 **회신 전문**을 그대로 붙인다.
> 회신이 매우 길면 핵심 판정줄은 본문에, 전문은 접기(`<details>`)나 별첨 파일로 보존한다(요약 대체 금지).
```
▷ 총괄 → <이름(역할)> [브리프 원문]
◁ <이름(역할)> → 총괄 [회신 원문]
```

## 3. 결정·산출 (선택)
- <합의/결정/커밋/blocker open·close>
```

## 기록 원칙 — 원문(verbatim) 우선 (필수)

- **요약하지 않는다.** 프롬프트·위임 브리프·회신은 **입력/출력 원문 그대로** 기록한다. 요약은 원문을 대체하지 못하며, 필요 시 원문 **위에** 덧붙이는 한 줄까지만 허용.
- **유일한 예외 = 시크릿·PII·자격증명 값** — 이것만 마스킹하고 **위치·유형**으로 대체(예: `NUXT_SESSION_SECRET=<32B 난수 주입, 값 무노출>`). 나머지는 원문 유지.
- 시각을 모르면 순서(①②③)로 표기(시각 강제 아님).

## 기록 시점

- 각 세션은 **의미 있는 프롬프트/위임/회신 발생 시** 자기 파일에 추가한다(세션 종료 시 일괄 정리도 가능).
- 허브는 배포·종합 보고 시 그날 `INDEX.md`를 갱신한다.
