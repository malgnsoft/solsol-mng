## 1. 이 패키지가 무엇인가

CreatorLMS(브랜드 쏠쏠) Figma 스토리보드 3영역(Customer Front / Customer Admin / Brand site)을 사내 표준(화면ID·UI/DEV 타입·헤더 7필드·Description 상태/예외·마스킹)으로 **증류한 텍스트 SoT**와, 그 시각 원본인 **참조 캡처(PNG)**, 그리고 **검증 가이드·체크리스트**를 한 묶음으로 만든 것입니다.

- 검증 단위 = **화면ID**(총 **272** = 본화면 194[Brand 39 + Front 48 + Admin 107] + 모달/팝업 78[FR 14 + AD 58 + BR 6], v1.2 3중검증 보정). 모달/팝업은 `부모화면ID_pu##`(공유 모달은 `S-{영역}-C##`)로 채번됨(00_화면목록 §3.1.2/§3.2.2/§3.3.2).
- 판정은 "퍼블 결과물 ↔ 참조캡처 PNG ↔ 화면설계서 본문" 3자 대조(상세 규칙은 00_검증가이드 §2 권위 분리).
- 증류는 1차 전수 완료 상태이며, 정책 충돌은 **05 정책설계서에서 16건(라운드1 6 + 라운드2 10) 운영자 컨펌 확정**되어 퍼블 판정 기준으로 들어가 있습니다.

> **긴 화면(tall)**: 세로로 긴 페이지는 `_exports/png/{영역}/pXXX.png` 한 장이 Read 시 다운스케일돼 소형 글자가 흐려질 수 있습니다. 그럴 땐 같은 폴더 `tall/pXXX_sNN.png`(≤1800px 밴드 슬라이스)가 풀해상도라 정확한 문구 판독에 씁니다.

### 06_API계약.md의 위치 — "참고", 검증 권위 아님
이 패키지의 1차 목적은 **퍼블리싱 UI 인터페이스 검증**입니다. 화면 판정 권위는 화면설계서·정책·참조캡처가 갖습니다(00_검증가이드 §2). **06은 프론트-백 API 인터페이스(엔드포인트·요청/응답·정책ID 매핑) 참고용**으로, 응답 마스킹(C-6)·알림 라우팅(M-4) 등이 어떤 계약으로 구현되는지 맥락이 필요할 때만 펼쳐 봅니다. 06 v0.2는 일부 OQ·크레딧 단가가 Open 상태입니다(아래 §3).

## 3. 알아둘 정책 상태 (검증 오탐 방지)

- **충돌 확정 16건**: 닉네임 2~15자(C-1), 무료체험 미운영(M-1), 강사 RBAC(M-2), 인증 유효시간(C-2), 비번 3종 8~16자(C-3), 쿠폰 정액 only(C-4), 토스트 일원화(C-5), 본인 화면 마스킹 분기(C-6), Admin 알림센터(M-4), Brand 마이페이지 접근주체(M-10) 등. 퍼블이 이 값을 안 따르면 결함(상세는 00_검증가이드 §4).
- **M-3 종량제 = 모델만 확정·단가 Open**: 크레딧 차감 과금 모델은 종량제로 확정됐으나 **단가(크레딧 수)·환산율은 입력데이터 수집 후 확정 예정으로 현재 Open**입니다. 따라서 **단가 수치가 비어 있는 게 정상**이고, 임의 단가가 박혀 있으면 오히려 결함입니다. **검증 담당은 단가를 추정·날조하지 마세요.**
- **`[추정]` 표기**: 화면설계서 본문 텍스트는 Figma vision-OCR 산물이라 일부 `[추정]`이 있습니다. 퍼블 문구가 본문과 달라도 곧바로 결함으로 단정하지 말고 **참조캡처 PNG(원본 렌더)를 확대해 확인**합니다(00_검증가이드 §3).
