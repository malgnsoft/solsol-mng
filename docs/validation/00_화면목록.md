## 1. 화면ID(Screen ID) 체계 — 필수

사내 표준 `DOCS-화면설계서작성표준.md` §2를 따른다.

**형식**: `S-[업무구분코드]-[Depth]-[화면번호][팝업구분]`
**예시**: `S-FR01-010102-001P01`

| 토큰 | 의미 | 규칙 |
|------|------|------|
| `S` | 화면설계서 고정코드 | 항상 `S` |
| `[업무구분코드]` | 영역(업무) 구분 | 아래 표 참조 (`FR01`/`AD01`/`BR01`) |
| `[Depth]` | 메뉴 계층 경로 | 2자리씩 메뉴 depth를 누적(예: `0101`=1depth 01 > 2depth 01) |
| `[화면번호]` | 화면 일련번호 | 3자리(`001`부터) |
| `[팝업구분]` | 해당 화면 내 팝업 | `P01`, `P02` … (없으면 생략) |

### 업무구분코드 (영역) 정의

| 코드 | 영역 | 설명 | 화면설계서 파일 | 캡처 폴더 |
|------|------|------|----------------|-----------|
| `FR01` | Customer Front | 수강생/고객용 학습 프론트 | `01_customer-front.md` | `screenshots/customer-front/` |
| `AD01` | Customer Admin | 고객(운영자/강사)용 관리자 | `02_customer-admin.md` | `screenshots/customer-admin/` |
| `BR01` | Brand site | 브랜드/마케팅 사이트 | `03_brand-site.md` | `screenshots/brand-site/` |

### 부여 규칙 (사내 표준)
- **★ Alert·Confirm(MPU)은 화면ID 미부여** — 모화면에 종속된 일시 모달이므로 ID를 부여하지 않는다.
- **PU/LPU/BS 등 독립 화면형 팝업은 화면ID 부여**(팝업구분 `P01` 사용).
- 화면ID는 **단계 간 추적 키**다 — `00_화면목록` ↔ `01~03 화면설계서` ↔ `04_정책요약`에서 동일 ID로 1:1 매핑(누락 0).

## 2. UI / DEV 타입 범례 (표 작성용)

- **UI TYPE**: `P`(전체화면,[이전]) · `PU`(전체팝업,[닫기]) · `LPU`(레이어 팝업) · `MPU`(Alert/Confirm 모달) · `BS`(바텀시트).
- **DEV TYPE**: `N`(네이티브)·`NI`(iOS)·`NA`(Android)·`W`(웹/하이브리드). CreatorLMS는 웹 기반 → **대부분 `W`**.
- 표의 `UI/DEV타입` 컬럼은 `P / W` 형식으로 함께 표기.
