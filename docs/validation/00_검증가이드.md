## 1. 30초 요약 — 무엇을 하나

1. **검증 단위 = 화면ID**. `00_화면목록.md`가 마스터 인덱스(**총 272 = 본화면 194 + 모달/팝업 78**[FR 14·AD 58·BR 6], v1.2 3중검증 보정), 상세 동작/정책은 `01~03`에 있습니다. 모달/팝업도 독립 검증 단위(`부모화면ID_pu##` / 공유 모달 `S-{영역}-C##`)로 채번돼 있으니 본화면뿐 아니라 모달/팝업도 클릭으로 띄워 인터랙션 대조(아래 §5 3번)하세요.
2. 각 화면을 **퍼블 결과물 ↔ 참조캡처(`_exports/png/{영역}/pXXX.png`) ↔ 화면설계서 본문** 3자로 대조합니다.
3. 결과를 **`01_검증체크리스트.md`** 표에 기록합니다(증거 경로 필수).
4. 판정: ⭕합격 / 🔺조건부(경미) / ❌재작업. **❌ 0건 = 라운드 종료 조건.**

## 2. 권위 분리 — "무엇을 무엇으로 검증하나" (오탐 방지 핵심)

축마다 **판정 기준(권위)이 다릅니다.** 이걸 섞으면 스펙의 한계를 퍼블 결함으로 잘못 보고하게 됩니다.

| 검증 축 | 판정 기준(권위) | 비고 |
|--------|----------------|------|
| 화면 존재/누락, IA·라우팅·active | **화면설계서**(00~03) | 화면ID 1:1 |
| 상태(빈/로딩/에러/예외) | **화면설계서** DESCRIPTION | 행복경로 외 상태 포함 |
| 공통 컴포넌트 일관성 | **화면설계서** C-1~C-15(Admin)·C-1~C-9(Brand) | 1회 정의 → 전 화면 대조 |
| 동작·정책 규칙 | **04 정책요약 + 05 정책설계서** | 05가 충돌건 최종값 |
| 개인정보 마스킹 | **화면설계서 마스킹 표기 + 작성표준 §7** | 카드 뒤4자리 등 |
| **정확한 문구/카피** | **참조캡처 PNG**(`_exports/png`) | PNG=Figma 원본 렌더 → 확대해 직접 읽음(문서 OCR 아님). §3 참조 |
| **색·간격·타이포(디자인 토큰)** | **스타일가이드**(문서) | 퍼블에 이미 적용됨 |
| 최종 비주얼 디테일 | **참조캡처 PNG + 스타일가이드** | 화면설계서엔 토큰 수치 없음 |

> 한 줄 원칙: **화면설계서 = "무엇이·어떻게 동작/상태/정책", 참조캡처 PNG = "정확한 문구·최종 비주얼", 스타일가이드 = "토큰 수치".**
> **Figma는 필수가 아닙니다.** PNG는 Figma 프레임을 export한 원본 이미지라, 검증의 시각·문구 권위는 PNG가 1차로 가집니다. Figma는 §3-바의 **예외 폴백**(아래)에서만 엽니다.

## 3. `[추정]` / OCR 텍스트 처리 규칙 (반드시 숙지)

- 화면설계서 본문 텍스트는 Figma **vision-OCR**로 추출한 것이라 **원본 문자열이 아닙니다.** 특히 `[추정]` 표기(주로 빈/로딩/에러 상태, Admin에 다수)는 캡처에서 명확히 확인 안 된 부분입니다.
- **규칙**: 퍼블의 문구가 화면설계서 **본문 텍스트**와 다르면 → 그건 *문서(OCR)가 틀렸을 수 있음*이므로 **"퍼블 결함"으로 단정하지 않습니다.** 대신 **참조캡처 PNG를 확대해 원문을 직접 확인**합니다(PNG=원본 렌더가 정답).
- **PNG로도 안 읽히는 경우만** Figma를 폴백으로 엽니다. 그조차 막히면 체크리스트 증거란에 `Figma확인요`로 분리(결함표 본문 아님).
- 즉 카피 판정 우선순위: **① 참조캡처 PNG(확대) → ② (글자 안 읽힘) Figma 폴백.** 문서 본문 OCR은 참고용일 뿐 권위가 아닙니다.

### 3-바. Figma 폴백이 필요한 좁은 경우 (그 외엔 PNG로 충분)
1. 색·간격·폰트의 **정확한 수치(hex/px)** 픽셀 정밀 비교가 필요할 때 — 단, 보통은 **스타일가이드 문서**에 값이 있어 Figma 불요.
2. PNG에 **안 찍힌 상태/프레임**이 의심될 때(갭 확인).
3. 확대해도 **글자가 안 읽히는** 한두 곳.
> 위 3가지 외에는 Figma 없이 PNG만으로 검증 가능합니다(전 영역 프레임 export 완료: Front 369·Admin 759·Brand 150).

## 6. 검증 워크플로우 (권장 — 캡처 하드게이트)

1. **준비**: 퍼블 화면을 브라우저로 띄울 수 있게 한다(정적 HTML 폴더 또는 로컬 URL).
2. **정적 대조**: 화면ID 순서대로 — 퍼블 화면 vs `_exports/png/{영역}/pXXX.png`(긴 화면은 `tall/pXXX_sNN.png` 슬라이스가 풀해상도) vs 화면설계서 본문.
3. **인터랙션 대조**(정적만으론 반쪽): 모달 열고-닫기, 드롭다운, 탭/필터, 모바일 햄버거 드로어, active 하이라이트 — **클릭으로만 드러나는 상태**를 직접 확인(QA 표준 §9.2).
4. **빠뜨리기 쉬운 상태 의도적 확인**: 빈 상태 / 에러·404 / 권한별 화면(RBAC) / 반응형(PC 1440×900·모바일 390×844) / 스크롤 하단.
5. **기록**: `01_검증체크리스트.md`에 축별 판정 + **증거(퍼블 캡처 경로 또는 참조 pXXX)**. 카피 불일치는 §3 규칙으로 분리.
6. **발산형 1축**(라운드당 최소 1회): "이 화면 구조가 애초에 맞나"도 1건 이상 본다(QA 표준 §8) — 결함과 별도 "구조 제언"으로 기록.

> 캡처 도구가 있으면 Playwright(headless, chromium)로 전 화면 fullPage 캡처를 떠 비교하면 누락이 줄어듭니다.

>
> 함께 받아야 완전: **퍼블 화면 접근 경로(폴더/URL)** — 검증 대상이라 패키지에 없으니 전달자에게 요청하세요.
