## C. 상급 Blocker 상세 (4건 — 종료조건 위반)

### ①·②·③ AD01 보안 (security-reviewer) — 보안 게이트 NO-GO
> ⚠️ 배포 `solsol-admin.pages.dev`는 지식베이스상 **의도된 '무인증 공개 미리보기(비프로덕션)'** 다. ①은 그 성격상 열려 있는 것이나, **프로덕션 전환 전 반드시 차단**해야 하며 ②는 미리보기 여부와 무관한 실제 시크릿 노출이다.

- **① demo-login 백도어(라이브 실증)** — `POST /api/auth/demo-login`이 자격증명·Origin 없이 `role:owner` 세션 쿠키 발급(HTTP 200). `previewDemo:'true'` 커밋 기본값이 활성. → 프로덕션 전 `NUXT_PREVIEW_DEMO=false` + 커밋 기본값 `'false'` + 상태변경에 Origin 필수화. `server/api/auth/demo-login.post.ts`·`nuxt.config.ts`.
- **② seed 관리자 자격증명 평문 하드코딩** — `nuxt.config.ts` L45–46(이메일+비번). 서버 전용이라 클라 번들엔 없으나 **git 이력에 실 dev-테넌트 owner 비번 잔존**. → `wrangler secret`/Pages env 이관·소스 제거·**노출 비번 로테이트**.
- **③ RBAC 라우트 가드 부재(EXC-06)** — 전역 미들웨어가 `isAuthenticated`만 검사, menuKey 페이지가드 없음 → 강사/서브강사 `/admin/settlement`·`/marketing`·`/users` 직접 URL 진입 시 셸 렌더(데이터 차단은 solsol-api 서버 인가 의존). → 클라 route 미들웨어 `menuKey→hasMenu` 가드 + solsol-api 서버 인가 재점검.
- **양호 실측**: 미인증 프록시 401 · SSRF 하드닝 견고(whitelist·`..`·메서드 405) · role은 서버 세션(`/api/admin/me`)만으로 결정(클라 위조 없음) · 토큰 httpOnly·Secure.

### ④ BR01 M-1 무료체험 문구 잔존 (qa) — 정책 정면 위반
- `index.vue`·`platform-intro.vue`·`pricing.vue`에 **"14일 무료체험" 배지·문구 잔존**(05 M-1 미운영 확정 위반, 존재하지 않는 체험 광고 = 오인·CS·법적 리스크). → 3개 LP에서 전면 제거(즉시결제 흐름). **근본원인 = 공통 셸(GNB/Footer) 컴포넌트 부재로 문구가 여러 파일에 산재**.
