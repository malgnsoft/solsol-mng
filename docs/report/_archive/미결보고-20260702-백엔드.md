## B. 미결 — 오너/정책 결정 대기

- **결제 실연동 전면 보류**: 양 백엔드의 결제·구독·환불·정산·크레딧충전·카드등록·캠페인발송·프로비저닝이 전부 **staging 게이트(골격+TODO)**. 실 토스/PG 연동은 오너 확인 후 해제.
- **브랜드 사이트 사용량 소스 미확정**: `my-sites`의 영상용량·수강신청 수치가 master에 없음 → 현재 0/mock. (tenant 집계 / solsol-api 집계 API / master 캐시 컬럼 중 택1)
- **사이트 프로비저닝 방식 미확정**: 사이트 생성 시 실제 테넌트 스키마 생성이 동기(brand-api 직접) vs 비동기(배치/워커) 미정 → 현재 TB_SITE·PROVISION_LOG 기록까지만.
- **정책 미확정**: 구독 플랜변경 비례정산 계산식, 연장 할인율, 카드삭제 시 활성구독 재바인딩, 브랜드 탈퇴 시 개인정보 파기 vs 법정보존(거래기록) 분리.
- **subjectType·payment.method 코드표**, 전용 **멱등 테이블**(uk) 신설 여부(현재 활성행 프록시로 대체 → 동시요청 레이스 잔존).

## C. 미결 — 실연동/후속 구현 (결제 라운드 GO 조건 포함)

- **refresh 세션원장 부재**(brand-api): stateless 서명 → 로그아웃 후에도 TTL(14일) 내 재발급 가능, 재사용 탐지 없음 → **TB_SESSION 도입 필요**.
- **토스 웹훅 서명검증 완성**: 양 백엔드 HMAC 골격만(헤더·canonicalization TODO) → 완성 전 상태전이 활성 금지.
- **R2 미바인딩**: 업로드(게시판 첨부·이미지, 수료증 PDF, 정산 통장사본, 브랜드 문의 첨부)가 메타/골격만.
- **라이브 채팅**(solsol-api): 인메모리 링버퍼(Workers 인스턴스 격리) → **채팅 테이블 신설 필요**.
- **이메일 발송 미연동**: 인증코드·비번재설정 메일이 `// TODO(mail)` 골격.
- **레이트리밋 미적용**: 로그인·인증코드발급·결제재시도 등 → Cloudflare WAF/Rate Limiting(인프라).
- **시크릿 미주입**(brand-api): `ALLOWED_ORIGINS`(프론트 CORS)·토스키 — 미설정이 안전 기본값이나 프론트 연결·결제 시 필요.

## D. 미결 — 정합/경미 (비차단)

- **동의 이력**(solsol-api): 주문 `agreed_terms`가 `TB_ORDER_LOG`에만 기록 → `TB_USER_AGREEMENT` 이관 권고. 브랜드 약관버전 `'1.0'` 하드코딩 → CMS 연동.
