## 6. 양호 확인 (근거 기반 — 공통)
- **SSRF 하드닝**: 프록시/쓰기 핸들러 화이트리스트 `admin/*`·`..`/스킴/인코딩슬래시 차단·URL 오리진 재검증(프로브: traversal 404·스킴 400·비-admin 404).
- **CSRF·메서드 분리**: 교차오리진 write/login/demo → 403. GET프록시↔write 405 상호차단.
- **세션/토큰**: httpOnly+Secure+SameSite=Lax(`admin_at`·`admin_rt`), 토큰·seed 비번 **클라 번들 미유출**, 새로고침 복원=서버 세션(`/api/admin/me`)만.
- **RBAC(M-2)**: LNB hide+프리셋+PermissionModal 심층방어(보조) + **서버 roleGuard 3단 최종강제**(requireAuth→requireUserType('staff')→roleGuard, TB_ADMIN_PERMISSION allowed/data_scope·EXC-06 403). 06 §2.8 정합.
- **비가역 게이트(CMP-06)**: 자동 환불 없음·파괴적 액션 컨펌/경고·write 자동실행 없음.
- **개인정보**: 타 회원 조회 name/email/phone **백엔드 선마스킹**(QA D03 payload 우려 name/email/phone은 해소) + 프론트 재마스킹(안전방향). 본인화면 C-6 정합. console PII 0.
- **백엔드(prior)**: PBKDF2(salt·constant-time)·refresh 회전+재사용탐지·테넌트 격리·IDOR 차단·SQLi 안전.
