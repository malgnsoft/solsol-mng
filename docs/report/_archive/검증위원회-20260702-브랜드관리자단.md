## 4. Blocker (상) — 배포 차단, 필수 해소

### SEC-1 · 크레딧 조정 멱등·원자성 결함 (`brand-api/src/routes/admin/credits.ts`)
- **현상**: ① 멱등 가드가 SELECT→INSERT **비원자**(Hyperdrive 읽기 캐시 stale) → 재전송·더블클릭 시 이중 조정 미검출. ② DB `uk(site_id, idempotency_key, source_credit_id)`가 검토 마이그레이션에 부재·Drizzle 정의도 주석뿐(존재 미확인)이고, 존재해도 credit 방향은 `source_credit_id=NULL` → **MySQL UNIQUE가 NULL을 서로 구별 → 동일 키라도 이중 지급 차단 불가**. ③ 차감이 openLots·balance를 **트랜잭션 밖·행잠금 없이** 읽음 → 동시 차감 시 lot 이중 소진·음수 잔액 실현(`FOR UPDATE` 부재).
- **영향**: 이중 크레딧 지급/차감, 원장 balance 훼손, 음수 잔액.
- **권고**: 별도 멱등 테이블 `uk(site_id, idempotency_key)`(NULL 미포함) **INSERT-first(중복=409/기존반환)** 또는 사이트 잔액행 `SELECT … FOR UPDATE`로 직렬화 + 잔액·lot 소진 전 과정을 **단일 트랜잭션+행잠금**. 담당: api-developer + dba.
