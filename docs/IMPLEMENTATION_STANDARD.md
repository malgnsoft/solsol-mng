# 구현 표준 — 목업 · 실앱(사용자단·관리자단) 계층·데이터 정책

> **작성**: 2026-07-02 (KST) · 총괄(chief) · **오너 확정**. 전 세션 공통 준수.
> 관련: [DEV_VALIDATION_PROCESS.md](DEV_VALIDATION_PROCESS.md)(검증) · [DEV_SESSION_PROTOCOL.md](DEV_SESSION_PROTOCOL.md)(세션).

## 0. 한 줄 원칙
**목업 = 가짜 데이터로 "디자인·기능"을 확인. 실앱(사용자단·관리자단) = 진짜 API·DB 연동으로 "실제 구현"을 확인.** 둘은 같은 화면ID로 1:1 연결된다.

---

## 1. 3계층 표준

| 계층 | 목적 | 데이터 | 백엔드 | 검증 |
|---|---|---|---|---|
| **목업 (mockup)** | 디자인·UX·기능 흐름이 잘 도는지 확인 | **가짜(목업) 데이터** — 하드코딩 픽스처, DB 없음 | 없음(프론트만) | 화면 9축(존재·IA·상태·카피·토큰·반응형·인터랙션) |
| **사용자단·관리자단 (실앱)** | 실제 구현이 **DB와 실연동**돼 동작함을 확인 | **진짜 경로** — 실 API + 실 Aurora DB. 레코드는 **샘플 데이터**(운영 전) → 실 데이터(운영) | **실 API + Aurora** | 9축 + 실연동(계약·에러·상태) + 데이터 5축 |

> "가짜 vs 진짜"의 정확한 뜻:
> - **목업 = 가짜 데이터**: 화면에 값이 하드코딩됨. API·DB를 거치지 않는다.
> - **실앱 = 진짜 연동**: 데이터가 **실 API → 실 DB**를 실제로 오간다. 그 안의 레코드는 개발·검증 단계엔 **샘플 데이터**(안전한 가짜 값이되 실 DB에 시드됨), 운영 단계엔 실제 고객 데이터. **연동 경로는 항상 진짜다.**

## 2. 데이터 정책 (계층별 — 핵심 구분)

### 목업 데이터 (가짜)
- 하드코딩 픽스처/목 composable. 실 API 호출 없음.
- 디자인·상태(빈/로딩/에러)·인터랙션을 **데이터 걱정 없이** 확인하기 위한 것.

### 실앱 데이터 (진짜 연동 + 샘플/실 레코드)
- **항상 실 API + 실 Aurora DB**를 경유한다.
- **샘플 데이터**(개발·검증): 가짜지만 **형식은 실제와 동일**(예: 홍길동·`sample@example.com`·카드 뒤4자리·긴 텍스트·빈 목록 엣지케이스) → 마스킹·정합·페이지네이션까지 실제처럼 검증 가능. **실 PII·시크릿 금지.**
- **실 데이터**(운영): 실제 고객·주문·크레딧 등. 샘플에서 자연스럽게 전환(스키마 동일).
- **샘플 데이터는 `backend-db` 세션이 단독 소유**(시드 스크립트) → dev 테넌트 Aurora에 적용(DBA self-env).

### 목(mock) 폴백 정책 — **개발용 허용 + 'MOCK' 배지 명시** (오너 확정)
- 실앱에서 API/스키마 미비로 **일시 목 폴백**이 필요하면, **해당 화면·영역에 `MOCK` 배지를 반드시 표시**한다(무음 폴백 금지 — 장애·미연동 은폐 방지).
- **검증·데모·릴리스 게이트 = `MOCK` 배지 0건**(전 화면 실 연동 확인). 배지가 남아 있으면 실앱 게이트 미통과.
- 즉 목 폴백은 **개발 중간 단계에서만**, 최종은 실 API + 샘플 데이터로 대체.

## 3. 유기적 연결 (세 계층을 하나로)
1. **동일 화면ID = 추적키** — 목업 화면 = 실앱 화면 = 검증 정본 `00_화면목록`이 **같은 ID로 1:1**(누락 0).
2. **목업 = 디자인 SoT** — 실앱은 목업의 **1px·토큰·카피·상태**를 그대로 구현(검증 시 목업 대조).
3. **흐름** — 목업(디자인·기능 확정) → 실앱(같은 화면을 실 API+샘플데이터로 구현·목폴백 배지 제거) → 검증 게이트.

---

## 4. 세션별 실행 (현 4세션)
- **`backend-db`**: 스키마·데이터 정본 확정 → **샘플 데이터 시드셋 작성**(도메인별 대표 + 엣지케이스) → dev Aurora 적용 → **API 계약 발행(/doc)**. *샘플 데이터 단독 소유.*
- **`solsol` / `solsol-brand`**: ① 목업은 **가짜 데이터로 유지**(디자인·기능 확정) → ② 실앱을 **실 API + 샘플 데이터로 연동**, **목 폴백엔 `MOCK` 배지**, 실패 시 에러/빈 상태 노출 → ③ 검증 시 배지 0건까지.
- **허브(hub)**: 화면ID 커버리지·검증 게이트·`_ledger` 집계로 **목업↔실앱 정합** 추적.

---

## 5. 검증 게이트 차등 (DEV_VALIDATION_PROCESS 정합)
- **목업 라운드**: 화면 9축(디자인·카피·토큰·상태·반응형·인터랙션) — 가짜 데이터 전제.
- **실앱 라운드**: 9축 전체 + **실연동**(계약·에러·상태) + **데이터 5축** + **샘플 데이터로 E2E**(조회·쓰기). 종료조건: **blocker 0 + `MOCK` 배지 0건**(무음 폴백·미연동 은폐 금지).
