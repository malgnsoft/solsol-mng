# validation_modified — 검증 정본 개정본 폴더

검증 시스템 정본([`../validation/`](../validation/))의 어떤 파일도 **직접 수정하지 않는다**(원본은 읽기전용 스냅샷으로 영구 보존).
정본 내용을 바꿔야 할 때는 **이 폴더에 동일한 파일명으로 개정 완성본(전체 MD)** 을 만들고, 변경 전후를 [`_개정이력.md`](_개정이력.md)에 기록한다.

## 규칙

1. **원본 무수정** — `docs/validation/<파일>` 은 절대 편집·삭제하지 않는다.
2. **개정본 = 동일 파일명, 완성본** — `docs/validation_modified/<같은 파일명>` 에 **개정 내용을 반영한 전체 문서**(부분 diff 아님, 그대로 열어 읽는 완성본)를 만든다.
3. **개정이력 별도 기록** — 개정할 때마다 [`_개정이력.md`](_개정이력.md) 에 **파일·rev·일자·사유·승인자 + 변경 전후**를 남긴다.
4. **승인 필수** — 정본 개정은 **오너/기획-lead 승인** 후에만(검증 절차 정본 [`../DEV_VALIDATION_PROCESS.md`](../DEV_VALIDATION_PROCESS.md) §0). 승인 없이 개정본을 만들지 않는다.

## 권위 (authority)

- **개정본이 존재하면**, 해당 화면·정책의 **현행 정본 = `validation_modified/<파일>`** 이다(승인 완료 기준). 원본 `validation/<파일>` 은 개정 전 스냅샷(이력용).
- 개정본이 없는 파일은 원본 `validation/<파일>` 이 그대로 정본이다.
- 검증·개발 시 우선순위: **개정본(있으면) → 없으면 원본**. 권위 순서(05→00~03→04→PNG→토큰)는 동일하게 적용.

## 사용 절차

1. 개정 필요 발견 → **오너/기획-lead 승인**.
2. 원본을 복사해 `validation_modified/<같은 파일명>` 생성 → 개정 내용 반영(완성본).
3. `_개정이력.md` 에 rev 항목 추가(사유·승인자·**전후**).
4. 필요 시 파생 산출물(ERD·screenList 등) 동기화는 소유 세션이 처리.

> 파일명은 원본과 **정확히 동일**하게 유지한다(예: 원본 `validation/05_정책설계서.md` → 개정본 `validation_modified/05_정책설계서.md`). 추적키(화면ID·정책ID)도 그대로 유지.
